## 3. 詳細実装計画

### タスク1: Googleフォームの設定
**優先度**: 🔴 最高  
**工数**: 1-2時間  
**担当**: 開発者 + クライアント

**作業内容**:
1. Googleフォームの作成
   - フォーム項目の設計（お名前、会社名、メール、電話、お問い合わせ種別、内容）
   - フォームのデザイン調整
2. 埋め込みコードの取得
   - iframe埋め込みURLの取得
   - またはリンクURLの取得
3. HTMLへの実装
   - `index.html`の`#contact`セクションを更新
   - プレースホルダーを実際のフォームに置き換え
4. 動作確認
   - フォーム送信テスト
   - レスポンシブ表示確認

**成果物**:
- 動作するお問い合わせフォーム
- フォーム送信先の設定完了

---

### タスク2: 連絡先情報の更新
**優先度**: 🔴 最高  
**工数**: 30分  
**担当**: 開発者 + クライアント

**作業内容**:
1. 連絡先情報の収集
   - メールアドレス
   - 電話番号
   - 営業時間
   - 住所（必要に応じて）
2. HTMLの更新
   - `index.html`の`#contact`セクション内の`.contact-info`を更新
   - ダミー情報を実際の情報に置き換え
3. 構造化データの更新
   - `index.html`内のJSON-LD構造化データに連絡先情報を追加

**成果物**:
- 正確な連絡先情報が表示されるページ

---

### タスク3: 実績画像の追加（最低限）
**優先度**: 🔴 高  
**工数**: 2-3時間  
**担当**: 開発者 + クライアント

**作業内容**:
1. 実績画像の準備
   - 最低3枚の実績画像を用意（推奨サイズ: 1200x800px）
   - 画像の最適化（WebP形式推奨、JPEGでも可）
2. 画像フォルダの作成
   - `images/works/`フォルダを作成
   - 画像ファイルを配置
3. HTMLの更新
   - `index.html`の`#works`セクションを更新
   - `.work-placeholder`を実際の画像に置き換え
4. CSSの調整（必要に応じて）
   - 画像表示のスタイル調整
   - レスポンシブ対応の確認

**成果物**:
- 実績セクションに実際の画像が表示される

**注意事項**:
- 画像の著作権確認
- 画像のファイルサイズ最適化（1MB以下推奨）
- 代替テキスト（alt属性）の適切な設定

---

### タスク7: Google Analytics導入
**優先度**: 🟢 中  
**工数**: 1時間  
**担当**: 開発者 + クライアント

**作業内容**:
1. Google Analyticsアカウントの作成
   - 測定ID（G-XXXXXXXXXX）の取得
2. HTMLへの実装
   - `index.html`の`<head>`セクションにGoogle Analyticsコードを追加
3. 動作確認
   - リアルタイムレポートでアクセスを確認

**成果物**:
- アクセス解析が可能な状態

**実装例**:
```html
<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>
```

## 5. リスク管理

### 6.1 リスク一覧

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|----------|------|
| 実績画像の準備が遅れる | 高 | 中 | プレースホルダーで公開し、後から追加 |
| Googleフォームの設定ミス | 中 | 低 | 送信テストを必ず実施 |
| 連絡先情報の誤り | 高 | 低 | 複数回の確認を実施 |
| 画像の著作権問題 | 高 | 低 | 使用許可を必ず取得 |

### 6.2 対策

1. **段階的公開**: 最低限の機能で公開し、後からコンテンツを充実
2. **テストの徹底**: 各機能の動作確認を必ず実施
3. **バックアップ**: 変更前にGitでバックアップを取得

## 7. 公開手順

### 8.1 GitHub Pagesでの公開

1. **リポジトリの準備**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin [リポジトリURL]
   git push -u origin main
   ```

2. **GitHub Pagesの有効化**
   - Settings → Pages → Source: main branch → / (root) → Save

3. **カスタムドメインの設定**（オプション）
   - Settings → Pages → Custom domain: `rogerfilm.zrst.jp`
   - DNS設定の更新

### 8.2 公開後の確認

- [ ] サイトが正常に表示される
- [ ] お問い合わせフォームが動作する
- [ ] モバイル表示が正常
